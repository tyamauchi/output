# 論文の概要 (Abstract)

- 5G商用化を経て、B5G(6G)ではより信頼性・速度・知能を備えた通信が求められる
- AI/MLはIoT・エッジ・クラウドまで5G/B5Gの基盤技術として不可欠
- 既存のセキュリティサーベイは**性能・精度**を重視し、**説明責任・信頼性**への言及が不足
- 本論文は **XAI (Explainable AI)** に着目し、ブラックボックス化したセキュリティ系AIモデルの透明性・説明可能性を検証
- ORAN、ゼロタッチ管理、E2Eスライシングなど全レイヤでのXAIの役割を整理
- 現行の研究プロジェクト・標準化動向・教訓・将来の研究方向性を提示

# なぜXAIが必要か:通信技術の脅威の変遷

| 世代 | 主な脅威 |
|---|---|
| 1G〜3G | 物理攻撃、盗聴、認証・認可の不備 |
| 4G | コア網へのDoS攻撃 [6]、SPIT、なりすまし、SIP登録ハイジャック [7] |
| 5G | SDN/NFV/クラウド化に伴う脅威(OpenFlow脆弱性、集中管理へのDoS、エッジ機器脆弱性、オープンAPI)[8][9] |

- 5G/B5Gの新技術(AI/ML、連携インテリジェンス)は**新たな脅威表面**を生む
- B5Gの成否はAIと6Gの協調にかかっている [10]

# ブラックボックスAIの課題とXAIの誕生

- Smart Cyber Physical Systems (SCPS) の相互接続増加 → 単一脆弱性が**バタフライ効果**的大規模障害を招く恐れ [11]
- AIをセキュリティに使う研究は長年存在するが [12][13]、AI自体をどう安全に運用するかは未解決
- **DARPA** が2017年5月に **XAIイニシアチブ** を開始 [14]
  - 目的: エンドユーザがAIを理解・信頼・管理できるようにする
  - 計算機科学と社会科学(人間心理学含む)の融合的取り組み
- B5Gの成功は、利用されるAIがどれだけ**頑健で信頼できるか**にかかっている [10]

# Section II: 5G/B5Gの基礎

- 5GはeMBB(高速ブロードバンド)を実現し、IoTの基盤を確立 [50]
- URLLC・mMTCにより、B5G(6G)ではデータレートが最大1Tbps規模に(現行比約1000倍)[17]
- 主要技術群:AI、Symbiotic Radio、Cell-free massive MIMO、RIS、SWIPT 等 [18][15][2][24]
- 膨大なデータ生成 → AIによる問題解決に最適な環境 [52]

§対応節: II-A / 関連Ref: [50][17][18][52]

# XAIの動機 (II-B1)

- DNNは数百層・数百万パラメータを持つ**洗練されたブラックボックス** [53]
- 重要な意思決定にブラックボックスMLが使われる場面が増加 → 説明可能性の需要増 [54]
- 医療分野の例:診断根拠を医師が把握することで信頼性向上 [55]
- XAIの効果:
  - モデルデバッグ支援・バイアス補正
  - 関連変数の妥当性確認、因果的推論の担保
  - セキュリティ監査・規制対応での有効性向上 [37][56]

# 図3:XAIタクソノミー(概念図)

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  Pre-model    │   │   In-model    │   │  Post-hoc    │
│  (学習データ説明)│   │ (透明モデル自体) │   │ (学習後モデル説明) │
│  PCA, t-SNE   │   │ 決定木, RF     │   │ LIME, SHAP    │
└──────────────┘   └──────────────┘   └──────────────┘
```

- **Pre-model**: 学習データを説明(特徴量エンジニアリング前)
- **In-model**: モデル自体が本質的に解釈可能
- **Post-hoc**: 学習済みモデルの出力を事後的に解釈

# III-A XAIによる緩和策

- LIME・SHAP・反実仮想説明でKPM(チャネル品質・スループット・遅延等)[91] を詳細に分析
- KPMが決定を反転させる**境界値**まで可視化 → ポイズニングの兆候を早期検知
- サンプル重要度ベースXAIでバックドア攻撃の**原因訓練サンプルを追跡**可能
- 課題:防御自体に使われるAE(オートエンコーダ)もまた攻撃対象となりうる [98]
  → 防御モデルにもXAIによる検証が必要

# III-C コア・バックホールネットワーク

- **NWDAF(Network Data Analytics Function)**が中核 [118]
  - AMF, SMF, AUSF, UDM, PCF, NSSF 等のNFがAIモデルを利用
- 脅威: 盗聴・DoS攻撃 [119]、TCPリセット・DDoS [121]
- 既存対策: 相互認証・鍵交換・IPsec/VPN方式 [119][120][121]
- ML活用例: Q学習によるmmW小セルバックホール信頼性向上 [122]、SDN上でのANNによるリソース再配分 [124]

§対応節: III-C1 / 関連Ref: [118][119][120][121][122][124]

---

# III-C XAIによる緩和策

- NWDAFのデータポイズニングをXAIで検知(モデルの意思決定シフトを追跡)
  - チュートリアル(II-C)でこの現象を実証
- **CBR(事例ベース推論)**をMLPに組み込み、障害発生時の責任所在を明確化 [126]
  - 復旧時間短縮・情報流出抑止に寄与
- サンプル重要度ベースXAIでバックドアの原因訓練データを特定 [127][128]

§対応節: III-C2 / 関連Ref: [124][126][127][128]

いずれも「脅威→XAIの役割→コスト→まとめ」の構成

<!-- Slide 35 -->
# IV-A E2Eスライシングとセキュリティ

- ネットワークスライシングは物理インフラを論理的に分割し、多様なサービス要件(URLLC/eMBB/mMTC)に対応
- **想定される脅威**: スライス間の分離不備によるサイドチャネル漏洩、スライス乗っ取り、リソース枯渇攻撃、AIベースのオーケストレーション層への攻撃
- **XAIの役割**: スライス割当・リソース制御AIの意思決定根拠を可視化し、異常なスライス挙動(越境アクセス等)の早期検知に寄与
- **課題**: マルチテナント環境での説明の粒度設計、リアルタイム性とのトレードオフ

§対応節: IV-A(概要:節タイトルおよび論文全体構成に基づく要約)

---

<!-- Slide 36 -->
# IV-B ネットワーク自動化・ZSM

- **ZSM(Zero-touch Network and Service Management)**は人手を介さない自律運用を目指す枠組み
- **想定される脅威**: 自動化ループ(closed-loop automation)そのものへの攻撃、誤ったポリシー生成、自律的意思決定の悪用
- **XAIの役割**: クローズドループ制御の各段階(監視・分析・意思決定・実行)で説明を組み込み、誤動作の原因追跡と説明責任を確保
- ゼロタッチ化が進むほど、**人間の監督が減る分だけXAIによる可視化の重要性が増す**

§対応節: IV-B(概要:節タイトルおよび論文全体構成に基づく要約)

# Section V: その他のセキュリティ実現技術とXAI

- **V-A** XAIと暗号化手法(データ難読化・暗号技術・匿名化)

§対応節: V Role of XAI on other security enablers in B5G

<!-- Slide 38 -->
# V-A 暗号化・匿名化とXAI

- **データ難読化 / 匿名化**: プライバシー保護のための前処理技術。GDPR等の規制対応と直結 [87]
- **暗号技術(準同型暗号HE・MPC等)**: 暗号化データ上でのAI推論は説明生成が困難という新たな課題を生む
- **XAIとの関係**: 暗号化・匿名化がモデルの入力特徴を隠すため、XAIの説明対象・粒度の再設計が必要
  - Differential Privacy(DP)適用時のノイズが説明の信頼性に影響する点も課題

§対応節: V-A / 関連Ref: [87]

---

<!-- Slide 39 -->
# V-B XAIと連合学習(Federated Learning)

- **連合学習**は生データを共有せず、分散端末上でモデルを協調学習する枠組み
- B5Gでは端末・エッジでのプライバシー保護学習として重要性が高い
- **XAIとの融合課題**:
  - 各クライアントのローカルモデルの説明をどう集約するか
  - グローバルモデルの説明がクライアント間で一貫するかの検証
  - 悪意あるクライアントによるモデル汚染(ポイズニング)をXAIで検知する余地

§対応節: V-B

---

<!-- Slide 40 -->
# V-C 将来ネットワークにおけるLLM向けXAI

- **LLM(大規模言語モデル)**は今後、ネットワーク運用・セキュリティ運用の自動化・対話型分析に活用が見込まれる
- Chain-of-Thought(CoT)等、LLM自身の推論過程の可視化技術がXAIの新たな適用領域
- **課題**: LLMの巨大なパラメータ空間・確率的出力に対する説明可能性の担保、ハルシネーションの検知・説明

§対応節: V-C / 関連(Table I 記載略語: CoT, LLMs)

---

<!-- Slide 41 -->
# V-D まとめ

- 暗号・匿名化・連合学習・LLMはいずれも「プライバシー保護」または「新たな知能化」を志向する技術
- これらとXAIの統合は、**セキュリティと説明可能性の両立**という共通課題を持つ
- B5Gにおいては、これら技術群が組み合わさって使われるため、統合的なXAI設計指針が求められる

§対応節: V-D Summary

# Section VI: XAI導入がもたらす新たなセキュリティ課題

4つの新規課題を提起:

1. **VI-A** 敵対的ML攻撃への脆弱性の増大
2. **VI-B** 新たな攻撃ベクトル・攻撃対象の出現
3. **VI-C** XAIを用いた安全なMLアプリケーション設計の難しさ
4. **VI-D** 倫理的配慮

§対応節: VI New Security Issues and Challenges of B5G Due to XAI

---

<!-- Slide 43 -->
# VI-A/B XAI自体が生む脆弱性

**VI-A 敵対的ML攻撃への脆弱性増大**
- 説明出力(特徴重要度等)が、逆に攻撃者にとってモデル内部を推測する手がかりとなりうる
- 「説明可能性 vs 攻撃耐性」のトレードオフが顕在化

**VI-B 新たな攻撃ベクトル・対象**
- XAIモジュール自体が新たな攻撃対象(説明の改ざん・操作)となる可能性
- 「説明のなりすまし」により、誤った安心感を利用者に与えるリスク

§対応節: VI-A, VI-B(節タイトルに基づく要約)

---

<!-- Slide 44 -->
# VI-C/D 設計上の難しさと倫理的課題

**VI-C 安全なML設計の課題**
- 精度・説明可能性・計算コストの三者トレードオフをどう設計するか
- 標準化された評価指標の欠如

**VI-D 倫理的配慮**
- 説明の公平性・偏り(バイアス)の検証責任
- GDPR等の規制と技術的実装のギャップ [87]
- 誰が「十分な説明」を判断する権限を持つかというガバナンス問題

§対応節: VI-C, VI-D / 関連Ref: [87]

---

<!-- Slide 46 -->
# Section VIII: 教訓と将来の研究方向(1/2)

**VIII-A XAIのB5Gセキュリティにおける役割**

- **教訓**: セクションIII〜VIを通じ、XAIはRAN・エッジ・コア・スライシング・自動化の全域で有効だが、リアルタイム性とのトレードオフが共通課題
- **未解決課題**: 計算効率の良いExplainable Edge-AI、スライス間の説明の分離・保護
- **予備的解決策**: 軽量XAI手法(サロゲートモデルの簡素化)、エッジオフロード戦略
- **将来方向性**: 標準化されたXAI評価指標の確立、ホワイトボックス/ブラックボックス双方での敵対的脆弱性評価

§対応節: VIII-A

---

<!-- Slide 47 -->
# Section VIII: 教訓と将来の研究方向(2/2)

**VIII-B 研究プロジェクトと標準に関する教訓**

- **教訓**: プロジェクト・標準化はまだ個別分散的で、XAI×セキュリティに特化した統合的枠組みが不足
- **課題**: 産学官の連携強化、心理学・社会学分野との協働不足
- **予備的解決策**: 既存標準化団体(3GPP/ETSI等)へのXAI要件の組み込み提案
- **将来方向性**: 6G標準化サイクルにXAI要件を初期段階から統合すること

§対応節: VIII-B

---

<!-- Slide 48 -->
# Section IX: 結論

- 本サーベイは、B5Gセキュリティの全レイヤ(RAN・エッジ・コア・スライシング・自動化)にわたる**XAIの役割**を包括的に整理した初の試み
- XAIは説明責任・透明性を通じてB5Gの信頼性向上に不可欠だが、**計算コスト・新たな攻撃面・倫理的課題**という代償も伴う
- 今後は、標準化団体との連携や軽量・堅牢なXAI手法の確立が、実運用への鍵となる
